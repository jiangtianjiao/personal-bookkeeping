# 个人记账项目 vs 开源方案 — 全面功能对比

> 生成日期：2026-04-08

---

## 一、核心记账能力

| 功能 | 本项目 | Firefly III | Actual Budget | ezBookkeeping | Beancount+Fava | GnuCash |
|------|--------|-------------|---------------|---------------|----------------|---------|
| **复式记账** | ✅ 完整（借贷平衡强制校验） | ✅ 完整（journal+transactions 两层） | ❌ 单式 | ❌ 单式 | ✅ 严格（余额断言验证） | ✅ 完整 |
| **多借多贷分录** | ✅ 支持 N 行 | ✅ 拆分交易 | ❌ | ❌ | ✅ | ✅ |
| **交易状态管理** | ✅ draft/posted/void | ✅ 类似 | ❌ | ❌ | ✅ (*/!/?) | ✅ (reconciled) |
| **金额精度** | ✅ 整数分存储 | ✅ decimal(32,12) | ✅ 整数分 | ✅ int64 | ✅ Decimal | ✅ GNC_Numeric |
| **账户层级** | ⚠️ 后端支持，前端未实现 | ❌ 扁平 | ❌ 扁平 | ✅ 两级账户 | ✅ 无限层级（A:B:C） | ✅ 无限层级 |
| **科目表体系** | 4 类（资产/负债/收入/支出） | 6 类（+Revenue+Reconciliation） | 无科目表 | 9 类细分 | 5 类（+Equity） | 5 类（+Equity） |

**结论**：本项目是少数支持完整复式记账的 Web 应用之一，与 Firefly III 处于同一水平。Actual Budget 和 ezBookkeeping 不支持复式记账。

## 十五、开源项目之间的差异化分析

### Firefly III 独有（其他都没有）

| 功能 | 说明 |
|------|------|
| **银行同步 (6000+ 银行)** | GoCardless/SaltEdge 集成，自动拉取银行流水，是最大竞争壁垒 |
| **Webhook 事件** | 交易创建/更新/删除时触发外部通知 |
| **Piggy Bank (存钱罐)** | 虚拟子账户，为特定目标存钱（如旅行基金） |
| **Bill 账单追踪** | 绑定周期性账单（如房租），自动匹配交易 |
| **Link (交易关联)** | 两笔交易之间建立关联关系（如退款关联原交易） |
| **JSON:API 规范** | 标准化 API 响应格式，便于第三方集成 |
| **独立 Data Importer** | 导入工具作为独立应用运行，不影响主系统稳定性 |

### Actual Budget 独有

| 功能 | 说明 |
|------|------|
| **本地优先架构** | 数据完全存储在用户设备，服务器仅用于同步 |
| **E2E 加密同步** | 跨设备同步时数据全程加密，服务器无法读取 |
| **信封预算法 (Envelope Budgeting)** | 零基预算核心理念，每一分钱都分配到信封 |
| **CRDT 同步引擎** | 基于冲突无关数据类型，支持离线编辑后无冲突合并 |
| **Goal Templates** | 为分类设置储蓄目标，自动计算每月需存金额 |
| **类原生 App 体验** | React 渲染速度接近原生应用 |

### ezBookkeeping 独有

| 功能 | 说明 |
|------|------|
| **20+ 导入格式** | 支持支付宝/微信/京东/随手记/Beancount/Firefly III/GnuCash/OFX/QIF/IIF/Camt.053/MT940 |
| **AI 票据识别 (MCP)** | 通过 MCP 协议接入 AI 模型识别收据图片 |
| **极致轻量 (30MB 内存)** | Go 语言编译为单二进制，资源消耗极低 |
| **WebAuthn/FIDO2** | 硬件安全密钥认证（YubiKey 等） |
| **地理位置记录** | 记录交易发生的经纬度坐标 |
| **交易图片附件** | 保存收据照片关联到交易 |
| **两级分类** | 一级分类（如"餐饮"）下设二级分类（如"外卖"、"堂食"） |
| **不接受捐赠** | 纯社区驱动，MIT 许可，永久免费无付费功能 |

### Beancount+Fava 独有

| 功能 | 说明 |
|------|------|

## 十六、核心结论

**本项目的独特优势**：
1. 同时支持复式记账 + 支付宝/微信导入的 Web 应用（Firefly III 有复式但无支付宝，ezBookkeeping 有支付宝但无复式）
2. 同技术栈中最现代的架构（React + TypeScript + Prisma + Service 层 + Zod）
3. 中文原生，不是翻译

**最大瓶颈**：
后端功能覆盖已达 Firefly III 的 60%，但**前端 UI 只暴露了约 40% 的后端能力**。标签、规则、预算、周期交易、导入这 5 个模块都是"后端已完成，前端未实现"状态。

**追赶路径**：
1. 优先补齐 5 个缺失的前端页面（可从后端直接获益，无需新增后端代码）
2. 多币种前端集成
3. Docker 部署
4. 邮箱账单自动导入（差异化竞争点，主流开源都没做）
