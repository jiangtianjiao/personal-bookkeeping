## Phase 4: 导入增强 ⚠️ 依赖最重的阶段

### 文件解析

| 格式 | 推荐方案 | 费用 | 说明 |
|------|---------|------|------|
| OFX/QFX | **自行解析**（正则 + 字符串） | 免费 | 现有 npm 包（ofx-js/node-ofx）均已停更，OFX 是简单 SGML 子集 |
| QIF | **自行解析** | 免费 | 纯文本行格式，解析不超过 100 行代码，现有库周下载 < 50 |
| Excel (XLS/XLSX) | `exceljs` | 免费 | MIT，5.5M 周下载，15k stars。SheetJS 社区版读取强但写入受限，Pro 需商业授权 |
| PDF 文本提取 | `unpdf` | 免费 | MIT，UnJS 生态维护，内置 PDF.js v5.4，替代已停更的 pdf-parse |
| 导入预览编辑 | Antd Table editable 模式 | 免费 | 项目已有 Antd，无需 AG Grid |

### 邮箱自动导入

| 组件 | 推荐方案 | 费用 | 说明 |
|------|---------|------|------|
| IMAP 客户端 | `imapflow` | 免费 | 473K 周下载，async/await 原生，支持 QQ/163/Gmail |
| 邮件解析 | `mailparser` | 免费 | Nodemailer 生态，同作者维护 |
| HTML 解析 | `cheerio` | 免费 | 容错性强，适合畸形邮件 HTML |

### AI 票据识别 ⚠️ 可能涉及付费

| 方案 | 每 1000 张成本 | 中文准确率 | 免费额度 | 推荐 |

## Phase 5: 自动化与智能化

### AI 对话记账 — 成本分析

| 方案 | 单次成本 (300 tokens) | 月成本 (日均 10 笔) | 推荐 |
|------|---------------------|-------------------|------|
| **GPT-4o-mini** | $0.00009 | **~$0.027（¥0.2）** | ★★★★★ 最便宜 |
| Claude Haiku 4.5 | $0.0007 | ~$0.21（¥1.5） | ★★★★☆ 质量更好 |
| **Ollama + Qwen2.5 7B** | 免费 | **免费** | ★★★★☆ 需 Mac 8GB+ |

**推荐**：GPT-4o-mini 为主（月成本可忽略），Ollama + Qwen2.5 作离线降级。
- SDK：`openai` npm 包
- 结构化输出：function calling 保证 JSON 格式
- 降级链：GPT-4o-mini → Ollama/Qwen2.5 → 规则引擎

**Qwen2.5 优势**：阿里开源，原生中文理解最强，Mac M 系列 30-60 tok/s。

## Gemini 3.1 Pro 替代建议（2026-04-08）

以下为 Gemini 3.1 Pro 对本选型方案的独立审查，核心理念是**压榨浏览器原生能力和 SQLite 内置功能，减少外部依赖**：

| 原方案 | 替代方案 | 理由 | 采纳建议 |
|--------|---------|------|---------|
| `@react-pdf/renderer` | CSS `@media print` + `window.print()` | 零依赖，复用现有组件 | ✅ 采纳：个人项目无需专业 PDF 库 |
| `sharp` | 前端 Canvas 压缩 | libvips 跨平台部署问题 | ✅ 采纳：避免 Docker 部署坑 |
| `fuse.js` | 后端 SQLite FTS5 | 前端不需内存全量搜索 | ✅ 采纳：减少一个依赖 |
| `natural` (NLP) | SQLite FTS5 BM25 相似度 | 查最相似历史交易取分类 | ✅ 采纳：零依赖且中文更好 |
| `exceljs` | `read-excel-file` | 更轻量，仅需读取 | ⚠️ 待评估：需确认 XLS(旧格式) 支持 |
| `node-cron` | DB 轮询表 (SQLite) | 进程重启不丢任务 | ✅ 采纳：更可靠 |

## 结论

**9 个阶段全部可以零成本或极低成本启动**。唯一涉及付费的是 Phase 4 的 OCR（Google Vision 1000 次/月免费）和 Phase 5 的 AI 对话（GPT-4o-mini 月均 ¥0.2）。所有核心功能均有纯免费开源方案可选。

**自行解析优于引入依赖**的场景：OFX、QIF、Beancount（格式简单，现有 npm 包质量差或已停更）。
