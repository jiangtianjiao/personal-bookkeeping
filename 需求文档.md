# 个人复式记账系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景
个人复式记账系统是一个面向个人用户的财务管理工具，基于复式记账原理（每笔交易借贷必相等），帮助用户精确记录每一笔收支、转账和投资，并提供直观的数据分析。

### 1.2 产品定位
- **目标用户**：有个人记账需求的普通用户
- **核心价值**：用专业的复式记账方法，以简单易用的方式管理个人财务
- **设计原则**：简单优先、专业但不复杂、本地数据为主

### 1.3 项目目标
- 实现完整的个人复式记账功能（借贷平衡）
- 提供快捷记账入口，降低日常记账门槛
- 提供直观的数据分析和可视化
- 支持多账户管理（现金、银行卡、信用卡等）
- 数据本地存储，保障隐私安全

## 2. 核心功能需求

### 2.1 用户认证
- 用户注册（用户名 + 邮箱 + 密码）
- 用户登录（邮箱 + 密码）
- JWT Token 认证
- 密码加密存储（bcrypt）

### 2.2 快捷记账
**2.2.1 快捷记账流程**
- 输入金额 → 选择分类（收入/支出/转账）→ 选择账户 → 填写备注 → 保存
- 系统自动生成借贷分录（用户无需理解复式记账原理）
- 支持默认账户设置，减少操作步骤

**2.2.2 生活化分类**
- **支出分类**：餐饮、交通、购物、居住（房租/房贷）、医疗、教育、娱乐、通讯、日用、其他
- **收入分类**：工资、奖金、兼职、投资收益、红包/礼金、其他
- **转账**：账户间资金转移

**2.2.3 智能辅助**
- 常用交易模板（如「早餐」「地铁」「月租」一键记账）
- 周期性交易自动记录（房租、工资等每月重复）
- 备注自动补全（根据历史记录推荐）

### 2.3 复式记账核心
**2.3.1 借贷记账法**
- 每笔交易必须同时记录借方和贷方
- 借贷必相等的原则校验（保存前强制校验）
- 支持一借多贷、一贷多借的复杂分录

**2.3.2 个人账户体系**
- **资产类**：现金、银行卡（借记卡）、投资账户
- **负债类**：信用卡、贷款、花呗/白条等
- **收入类**：工资、兼职收入、投资收益等
- **支出类**：餐饮、交通、购物等各项生活支出

**2.3.3 会计期间管理**
- 按月划分会计期间
- 月度结账功能（结账后该期间不可修改）
- 期初余额录入
- 反结账功能（撤销月度结账）

### 2.4 交易管理
**2.4.1 交易录入**
- 快捷记账界面（普通用户入口）
- 高级分录编辑器（专业用户入口，可自由编辑借贷分录）
- 模板交易功能
- 批量导入（CSV/Excel）

**2.4.2 交易编辑**
- 交易修改和删除（未结账期间）
- 交易状态管理：草稿（已结账期间）通过红字冲销修改
- 交易撤销（Draft 状态可直接删除）

**2.4.3 交易查询**
- 按日期范围筛选
- 按账户筛选
- 按分类筛选
- 按金额范围筛选
- 关键词搜索（备注内容）
- 按交易状态筛选（草稿/已过账/已冲销）

**2.4.4 交易状态**
- `draft` - 草稿，可自由编辑
- `posted` - 已过账，正式入账
- `void` - 已冲销，通过红字冲销标记

### 2.5 账户管理
**2.5.1 账户类型**
- 现金账户
- 银行卡（借记卡）
- 信用卡账户
- 电子钱包（支付宝/微信）
- 投资账户
- 贷款账户
- 其他自定义账户

**2.5.2 账户功能**
- 账户余额实时计算（从交易分录汇总，不单独存储余额字段避免不一致）
- 账户间转账
- 账户状态管理（启用/停用）
- 账户初始余额设置

### 2.6 数据管理
**2.6.1 数据存储**
- SQLite 本地数据库
- 数据文件可导出备份

**2.6.2 数据导入导出**
- 导出为 CSV/JSON 格式
- 从 CSV 导入交易记录
- 数据库文件整体备份和恢复

**2.6.3 数据校验**
- 借贷平衡校验
- 账户余额一致性检查
- 重复交易检测

## 5. 技术架构

### 5.1 前端技术栈
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **UI组件库**: Ant Design 5
- **图表库**: ECharts 5
- **状态管理**: Zustand
- **路由**: React Router 6
- **HTTP客户端**: Axios
- **日期处理**: Day.js

### 5.2 后端技术栈
- **框架**: Express.js + TypeScript
- **数据库**: SQLite
- **ORM**: Prisma
- **认证**: JWT (jsonwebtoken)
- **密码加密**: bcryptjs
- **输入验证**: express-validator
- **日志**: Winston（生产环境）

### 5.3 数据库设计要点
- 金额字段使用 `Decimal` 类型，避免浮点精度问题
- 账户余额通过交易分录实时计算，不单独存储
- 所有表使用 UUID 主键
- 软删除策略（重要数据不物理删除）
- 适当的索引优化查询性能

## 6. API 设计

### 6.1 认证
```
POST   /api/auth/register     - 用户注册
POST   /api/auth/login        - 用户登录
GET    /api/auth/profile      - 获取用户信息
```

### 6.2 账户管理
```
GET    /api/accounts          - 获取账户列表
GET    /api/accounts/:id      - 获取账户详情（含余额）
POST   /api/accounts          - 创建账户
PUT    /api/accounts/:id      - 更新账户
DELETE /api/accounts/:id      - 停用账户
GET    /api/accounts/:id/transactions  - 获取账户交易
```

### 6.3 交易管理
```
GET    /api/transactions          - 获取交易列表（支持筛选/分页）
GET    /api/transactions/:id      - 获取交易详情
POST   /api/transactions          - 创建交易
PUT    /api/transactions/:id      - 更新交易（仅 draft 状态）
DELETE /api/transactions/:id      - 删除交易（仅 draft 状态）
POST   /api/transactions/:id/post - 过账（draft → posted）
POST   /api/transactions/:id/void - 冲销（posted → void，生成红字冲销记录）
```

### 6.4 报表
```
GET    /api/reports/balance-sheet         - 资产负债表
GET    /api/reports/income-expense        - 收支报表
GET    /api/reports/trial-balance         - 试算平衡表
GET    /api/reports/dashboard             - 仪表板数据
```

### 6.5 分类管理
```
GET    /api/categories           - 获取分类列表
POST   /api/categories           - 创建分类
PUT    /api/categories/:id       - 更新分类
DELETE /api/categories/:id       - 删除分类
```

## 7. 非功能需求

### 7.1 性能要求
- 页面加载时间 < 2秒
- 交易录入响应时间 < 500ms
- 支持万级以上交易记录流畅查询

### 7.2 安全要求
- 密码 bcrypt 加密存储
- JWT Token 认证
- API 输入参数校验
- CORS 跨域配置
- 数据库文件本地存储，不上传云端

### 7.3 兼容性要求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 响应式设计适配移动端

3. 可视化图表（ECharts）
4. 预算管理
5. 数据导入导出（CSV）
6. 会计期间管理（结账/反结账）

### Phase 3: 体验提升
1. 移动端深度适配
2. 暗/亮主题切换
3. 快捷键支持
4. PWA 离线使用
